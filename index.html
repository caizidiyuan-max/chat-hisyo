説明・譲渡方法

1．このアプリの動き

「承認」はGoogleの接続許可ではなく、AI返信案の内容を本人が了承する操作です。承認すると名前・電話番号・メールを付けたGmail下書きを作ります。次の画面で宛先・件名・本文・CC・BCC・添付を確認し、「送信」を押した時だけGmailから送信します。

GitHub Pages版は公開用の画面見本です。実Gmailへ接続してアプリから送信する本体は、同じHTMLを本人専用のApps Script Webアプリとして開きます。

2．Gmail接続とGemini API

GitHub版へAPIキーを入力しないでください。公開ページなので秘密情報を保存できません。

本人専用Apps Script版の「署名・通知設定」を開くと、名前・電話番号・Gmail・Gemini APIキーを安全に設定できます。APIキーはApps Scriptの非公開プロパティへ保存し、画面へ再表示しません。GmailはApps ScriptのGoogle許可を使うため、Gmail用APIキー、OAuth Client ID、Client Secret、リダイレクトURIは不要です。

普段使う有料GeminiとGemini APIは別の利用枠です。AI Studioで請求設定をしていなければ、API無料枠を使います。

3．相手へ譲渡する手順

  1. index.htmlとCode.gsを渡す
    公開見本はGitHubへindex.htmlとして置けます。
  2. 相手が自分のAPIキーを作る
    元の所有者のAPIキーは渡しません。
  3. 相手が新しいApps Scriptを作る
    Apps Scriptで新規作成し、Code.gsと「index」というHTMLファイルを追加します。
  4. 本人専用Webアプリとしてデプロイ
    「自分として実行」「アクセスできるユーザーは自分」にします。
  5. 署名・通知設定を開く
    相手自身のAPIキー・名前・電話番号・Gmailを入力し、「保存してGmailを読み込む」を押します。
  6. Googleの権限を本人が許可する
    Gmail・Drive・Sheetsを扱う許可は相手自身のGoogleアカウントで行います。
  7. 下書きと送信履歴を確認
    承認では下書きだけ、最終送信後にGmail送信済みへ残ることをテストします。

4．容量の見方

AI Studioの使用量と上限画面で「Gemini 2.5 Flash」を確認します。分析・起草・校正の3役を分けるため、下書き1通で通常3リクエスト使います。混雑時は安全に止まり、5分後に再試行します。

5．安全チェック

  • APIキーをHTML・GitHub・チャットへ書かない
  • 相手ごとに別のApps ScriptとAPIキーを使う
  • 実メールをGitHubへ保存しない
  • 無料Gemini APIで機密メールを扱う場合はGoogleのデータ利用条件を確認する
HTMLの更新では、Gmail・Apps Script・Google Driveのバックアップは消えません。実データはHTMLとは別の場所にあります。

6．配布ファイル

実用版は同時に配布する最新の Code.gs を使ってください。秘密情報は含まれず、APIキーは利用者本人がスクリプトプロパティへ設定します。
